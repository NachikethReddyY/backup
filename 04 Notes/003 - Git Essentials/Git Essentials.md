# Git Essentials

## Git Data Storage

Git stores data through commits, which consist of:

- **Commit**: Contains metadata like author, log message, and commit time
- **Tree**: Tracks the names and locations of files and directories
- **Blobs**: Contain compressed snapshots of file contents

Commit Visualization: You examined three commits to understand how Git links files and directories to their respective blobs.

## Hashes

Unique 40-character identifiers generated by a hash function, allowing Git to efficiently manage and share data between repositories. For referencing, use the first 8-10 characters.

## Viewing Commit Information

Use `git log` to display all commits in reverse chronological order, showing the commit hash, author, date, and commit message.

```bash
git log
git log -3
git log report.md
git log -2 report.md
```

### Filter by Date

```bash
git log --since "Month Day Year"
git log --until "Month Day Year"
git log --since "Month Day Year" --until "Month Day Year"
```

You can use natural language: `"2 weeks ago"`, `"3 months ago"`, `"yesterday"`, or ISO format: `YYYY-MM-DD` (e.g., `07-15-2024`).

### Show Specific Commit

```bash
git show c27fa856
```

## Comparing Changes

### git diff

Shows the difference between commits. Compares the last committed version with the latest version that is not in the staging area.

```
@@ -1,5 +1,5 @@
```

This output format means file a starts from line 1 to 5 and file b also from line 1 to 5. The minus sign indicates a removed line, and plus indicates an added line.

```bash
git diff --staged report.md
git diff c27fa856 18639cf
git diff HEAD~1 HEAD
```

## Reverting Changes

### git revert

Creates a new commit that undoes a previous commit.

```bash
git revert --no-edit HEAD
git revert -n HEAD   # Without commit, move to staging area
```

### git checkout

Revert a single file to a previous state.

```bash
git checkout HEAD~1 -- report.md
```

### git restore

Remove a file from the staging area.

```bash
git restore --staged file
```

## Viewing Staging Area

```bash
git status
```
